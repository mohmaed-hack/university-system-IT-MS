# 🎓 نظام الإدارة المتكامل للموقع الأكاديمي

## نظرة عامة

هذا المشروع يوفر **نظام إدارة ويب متكامل واحترافي** للموقع الأكاديمي، مما يمكّن المشرفين من التحكم الكامل بجميع محتويات الموقع من خلال واجهة سهلة الاستخدام.

## 🚀 البدء السريع

### المتطلبات
- Node.js 18+
- pnpm (أو npm/yarn)
- حساب GitHub مع personal access token

### التثبيت

```bash
# 1. استنساخ المشروع
git clone <your-repo>
cd <project-folder>

# 2. تثبيت المكتبات
pnpm install

# 3. إعداد البيئة
cp .env.example .env.local

# 4. تشغيل المشروع
pnpm dev
```

### المتغيرات المطلوبة في `.env.local`

```env
ADMIN_PASSWORD=your-secure-password
GITHUB_TOKEN=ghp_xxxxxxxxxxxx
GITHUB_REPO=username/repo-name
GITHUB_BRANCH=main
```

### الوصول إلى الموقع

- **الموقع:** http://localhost:3000
- **تسجيل الدخول:** اضغط "المشرف" ← أدخل كلمة المرور
- **لوحة التحكم:** اضغط على شارة "المشرف" من الصفحة الرئيسية

## 🎯 الحالات الاستخدام

### لإضافة مادة جديدة:
1. تسجيل دخول إداري
2. الصفحة الرئيسية → إدارة المواد
3. اختر الترم → إضافة مادة
4. ملء البيانات
5. حفظ التغييرات

### لإضافة محاضرة:
1. اذهب إلى المواد → اختر مادة
2. انقر "إدارة المحاضرات"
3. إضافة محاضرة جديدة
4. ملء البيانات (العنوان، الروابط)
5. حفظ

### لإضافة اختبار:
1. لوحة التحكم → اختبارات السنة
2. اختر الترم
3. إضافة اختبار جديد
4. ملء البيانات الكاملة
5. حفظ التغييرات

## 🔐 الأمان

### كيفية عمل الأمان

1. **المصادقة**: كلمة مرور مع HMAC
2. **الجلسات**: HTTP-only cookies محمية
3. **التفويض**: فقط المشرفون يرون الأزرار
4. **API**: محمية بفحص الجلسة
5. **البيانات**: محفوظة بأمان في GitHub

### أفضل الممارسات

- استخدم كلمة مرور قوية (30+ حرف)
- غيّر كلمة المرور بانتظام
- لا تشارك البيانات الحساسة
- استخدم HTTPS على الإنتاج
